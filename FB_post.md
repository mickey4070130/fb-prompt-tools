# 安親班 FB 貼文模板說明書

## 文件目的
本文件用來定義安親班 Facebook 貼文的固定模板，讓後續系統可自動產生文案，同時維持自然、可信、有人味的語氣。

本文件重點：
- 建立可大量複用的文案骨架
- 確保語氣不像機器生成
- 讓不同活動（安親班活動、美語課活動、節慶活動）都能快速套用

---

## 使用原則（避免 AI 感）
### 1. 語氣像老師或班主任在分享
- 用「我們今天」「孩子們」「這堂課」等真實場景語言
- 句子長短交錯，不要整篇都同樣節奏
- 帶入可觀察細節（孩子反應、現場互動）

### 2. 避免過度制式語句
避免常見 AI 腔：
- 「在這個充滿挑戰的時代」
- 「讓我們一起」
- 「每一位孩子都是獨一無二」
- 連續重複「透過...提升...能力」

### 3. 每篇只講一個主軸
- 專業：聚焦「教學設計與成果」
- 痛點：聚焦「家長常見擔心與對應做法」
- 溫馨：聚焦「孩子互動、成長瞬間、情感連結」

### 4. 文字要能對上照片
- 照片是分組活動，就寫合作與討論
- 照片是美語課，就寫口說互動或任務
- 不寫照片看不出來的內容

---

## 貼文共用欄位（變數）
以下欄位供後續自動化系統填入：

- `{活動名稱}`：例如「美語闖關日」
- `{課程類型}`：例如「美語課 / 主題活動 / 安親日常」
- `{日期或週次}`：例如「本週三」「3 月第 2 週」
- `{年級或班別}`：例如「中年級班」
- `{活動亮點}`：例如「分組任務、單字配對、角色扮演」
- `{孩子表現}`：例如「主動舉手、願意開口、互相協助」
- `{教學目標}`：例如「口說勇氣、專注力、表達能力」
- `{給家長的訊息}`：例如「回家可一起練習 3 個句型」
- `{CTA}`：例如「歡迎私訊了解課程安排」
- `{Hashtags}`：例如「#安親班 #美語課 #學習日常」

---

## 文案長度規範
- 建議字數：80~180 字（一般貼文）
- 長文上限：250 字（活動成果回顧）
- 段落數：2~4 段為主
- 每段 1~3 句，避免一大段難閱讀

---

## 模板 A：專業風格（清楚、可信、重教學）
### 使用情境
- 課程成果分享
- 美語課程設計說明
- 讓家長理解「不是只寫功課」

### 結構
1. 開場：今天做了什麼課程/活動
2. 教學設計：怎麼帶、重點任務是什麼
3. 孩子反應與成果：看見哪些具體表現
4. 收尾：給家長一句實用提醒 + CTA

### 模板
今天在 `{課程類型}` 的 `{活動名稱}`，我們帶 `{年級或班別}` 孩子進行 `{活動亮點}`。  
課程設計的重點是 `{教學目標}`，不只完成任務，也要讓孩子在過程中真正理解、敢開口。  
從今天的表現可以看到 `{孩子表現}`，學習狀態很扎實。  
`{給家長的訊息}`。`{CTA}`  
`{Hashtags}`

### 語氣提示
- 多用具體詞：「分組討論」「輪流發表」「句型練習」
- 少用空泛詞：「全方位」「高效學習」「多元能力」

---

## 模板 B：痛點風格（先懂家長擔心，再給解法）
### 使用情境
- 家長常見問題貼文
- 招生期、詢問量增加時
- 想強調本班方法與差異

### 常見痛點可選
- 回家功課拖很久、親子衝突增加
- 英文學很久但不敢開口
- 孩子注意力容易分散
- 寫完作業卻不知道有沒有真正懂

### 結構
1. 痛點開場：家長熟悉的真實場景
2. 我們做法：在班上的實際安排
3. 今日案例：活動中觀察到的改變
4. 收尾：降低焦慮 + CTA

### 模板
很多家長會提到 `{痛點主題}`，這其實很常見。  
在 `{課程類型}` 的安排上，我們會用 `{活動亮點}`，把孩子從「被動完成」帶到「知道自己在學什麼」。  
以今天 `{活動名稱}` 來看，孩子出現了 `{孩子表現}`，進步是看得見的。  
`{給家長的訊息}`，如果你也在意這件事，`{CTA}`  
`{Hashtags}`

### 語氣提示
- 痛點要真實，但不要製造恐慌
- 重點是提出具體做法，不是只放大問題

---

## 模板 C：溫馨風格（有畫面、有情感、自然）
### 使用情境
- 日常活動記錄
- 節慶活動、成果發表
- 孩子互動瞬間、班級氛圍分享

### 結構
1. 畫面開場：今天的現場片段
2. 過程描述：孩子怎麼互動、嘗試
3. 成長點：小小進步與觀察
4. 收尾：溫暖一句 + CTA

### 模板
今天的 `{活動名稱}` 很熱鬧，`{年級或班別}` 的孩子在 `{活動亮點}` 時特別投入。  
有人一開始有點害羞，但在同學鼓勵下也願意試試看，現場可以看到 `{孩子表現}`。  
這些看起來小小的時刻，其實都在累積 `{教學目標}`。  
謝謝孩子們今天的認真，`{給家長的訊息}`。`{CTA}`  
`{Hashtags}`

### 語氣提示
- 可以有溫度，但避免過度抒情
- 用看得到的細節代替抽象感動

## 標題模板（可選）
每篇可搭配 1 句短標題：

- 專業風：`今天的 {活動名稱}，孩子不只完成，更學會方法`
- 痛點風：`孩子不是不會，是需要對的方法`
- 溫馨風：`今天又收集到孩子們一個很棒的成長瞬間`

## CTA 模板（行動呼籲）
- 想了解 `{課程類型}` 的安排，歡迎私訊我們。
- 想看孩子適合哪一種學習節奏，歡迎預約試讀。
- 有課後安排需求，歡迎留言或私訊聊聊。

## 產出檢查清單（發文前）
- 文案是否看得出今天實際做了什麼
- 是否有至少 1 個可被照片驗證的細節
- 語氣是否像老師真人分享，而非宣傳機器文
- 是否避免空泛、浮誇、過度模板感
- CTA 是否自然，不硬推銷

## 自動化輸入格式（MVP 版）
本節定義後續程式/API 的輸入資料格式。先用 JSON，之後可對接 Google Sheet 或表單。

### A. 必填欄位
- `post_style`：`professional` / `pain_point` / `warm`
- `activity_name`：活動名稱
- `course_type`：課程類型（美語課/主題活動/安親日常）
- `class_group`：年級或班別
- `activity_highlights`：活動亮點（字串陣列，1~5 筆）
- `child_observations`：孩子表現（字串陣列，1~5 筆）
- `teaching_goal`：教學目標
- `parent_message`：給家長的訊息
- `cta`：行動呼籲
- `hashtags`：Hashtag（字串陣列，3~6 筆）

### B. 條件必填欄位
- `pain_point_topic`：當 `post_style = pain_point` 時必填

### C. 選填欄位
- `date_label`：例如「本週三」「4 月第 1 週」
- `title`：自訂標題，未提供則系統依風格生成
- `photo_notes`：照片補充說明（避免文案與照片不一致）
- `tone_strength`：`light` / `normal` / `strong`（語氣強度）

### D. 欄位驗證規則
- `activity_name`：2~20 字
- `teaching_goal`：5~30 字
- `activity_highlights` 每筆：4~25 字
- `child_observations` 每筆：4~25 字
- `parent_message`：8~60 字
- `cta`：6~30 字
- `hashtags`：每筆需以 `#` 開頭

---

## 產出資料格式（建議）
程式回傳可用以下格式，方便前端或排程工具直接顯示：

```json
{
  "results": [
    {
      "post_style": "professional",
      "title": "今天的美語闖關日，孩子不只完成，更學會方法",

      "char_count": 146
```

## 產文流程規則（給系統）
1. 先驗證必填欄位與字數規則，不通過就回傳錯誤訊息。
2. 依 `post_style` 選模板，將陣列欄位轉為自然句（用頓號或逗號）。
3. 檢查是否出現 AI 腔禁語，若有則改寫。
4. 檢查是否至少含 1 個可對照片驗證的活動細節。
5. 回傳 `title + body + hashtags + char_count`。

## LLM Prompt 規格（可直接用）
本節提供可直接對接模型 API 的 Prompt 設計，目標是穩定輸出「不 AI、可發文」的內容。

### A. System Prompt（固定）
```text
你是安親班的資深班主任文案助手。你的任務是根據輸入資料，產生 Facebook 貼文。

必須遵守：
1) 語氣自然，像真人老師分享，不要像廣告機器文。
2) 內容務必可被活動照片驗證，不可幻想不存在的細節。
3) 禁止使用空泛口號與 AI 腔，例如「在這個充滿挑戰的時代」「讓我們一起」「每一位孩子都是獨一無二」。
4) 專業風、痛點風、溫馨風必須有明顯差異。
5) 文字以繁體中文輸出。
6) 只輸出 JSON，不要加任何前後說明。
```

### B. User Prompt（單篇模板）
```text
請依下列輸入資料生成 1 篇 FB 貼文。

[INPUT]
{json_input}

[RULES]
- post_style=professional：重點放教學設計與可觀察成果。
- post_style=pain_point：先點出家長痛點，再給具體做法，不能製造焦慮。
- post_style=warm：有畫面、有溫度，但避免過度抒情。
- 字數建議 80~180 字，最多 250 字。
- Hashtag 請組成單一字串，空格分隔。
- 若 title 未提供，依風格生成 1 句自然標題。
- body 必須至少包含 1 個可被照片驗證的活動細節。

[OUTPUT_JSON_SCHEMA]
{
  "post_style": "professional|pain_point|warm",
  "title": "string",
  "body": "string",
  "hashtags": "string",
  "char_count": 123
}
```

### C. User Prompt（三風格批次模板）
```text
請根據輸入資料，一次產生三種風格的 FB 貼文：professional、pain_point、warm。

[INPUT]
{json_batch_input}

[RULES]
- 三篇都要基於同一活動事實，不得虛構。
- 三篇語氣要有差異，不能只換幾個詞。
- 每篇 80~180 字。
- 每篇都要有 title、body、hashtags、char_count。
- Hashtag 以單一字串輸出（例：#安親班 #美語課 #學習日常）。
- 只輸出 JSON。

[OUTPUT_JSON_SCHEMA]
{
  "results": [
    {
      "post_style": "professional",
      "title": "string",
      "body": "string",
      "hashtags": "string",
      "char_count": 123
    },
    {
      "post_style": "pain_point",
      "title": "string",
      "body": "string",
      "hashtags": "string",
      "char_count": 120
    },
    {
      "post_style": "warm",
      "title": "string",
      "body": "string",
      "hashtags": "string",
      "char_count": 132
    }
  ]
}
```

### D. 失敗回傳格式（建議）
若欄位不足或格式錯誤，建議模型回傳：

```json
{
  "error": true,
  "error_code": "VALIDATION_ERROR",
  "message": "pain_point 風格缺少 pain_point_topic",
  "missing_fields": ["pain_point_topic"]
}
```

### E. 口吻微調參數（建議）
可在 user prompt 增加可控參數：
- `tone_strength=light`：更口語、更日常
- `tone_strength=normal`：平衡
- `tone_strength=strong`：觀點更明確，但仍自然

## LLM 腳本使用方式
### 檔案
- `generate_fb_post_llm.py`：呼叫 OpenAI Responses API 產文
- `examples/single_input.json`：單篇輸入範例
- `examples/batch_input.json`：三風格批次輸入範例

### 指令
先看 Prompt 組裝是否正確（不呼叫 API）：
```bash
python generate_fb_post_llm.py -i examples/single_input.json --dry-run
python generate_fb_post_llm.py -i examples/batch_input.json --dry-run
```

實際呼叫 API 前，先設定金鑰：
```bash
# PowerShell
$env:OPENAI_API_KEY="你的金鑰"
```

呼叫 API 並輸出結果：
```bash
python generate_fb_post_llm.py -i examples/single_input.json -o out_single.json
python generate_fb_post_llm.py -i examples/batch_input.json -o out_batch.json
```

## 一鍵產文（不用改 JSON）
### 檔案
- `quick_fb_post.py`

### 最快用法
直接輸入活動名稱，其他欄位走預設值，一次產生三種風格：
```bash
python quick_fb_post.py --activity-name 美語拼字挑戰
```

### 互動模式
不帶參數執行，系統會逐項詢問：
```bash
python quick_fb_post.py
```

### 指定單一風格
```bash
python quick_fb_post.py --activity-name 美語拼字挑戰 --style warm
```

### 輸出到檔案
```bash
python quick_fb_post.py --activity-name 美語拼字挑戰 -o out_quick.json
```
